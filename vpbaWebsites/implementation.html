Standard Implementation (Website Created by Site Owners)

The main idea of this implementation process is that we create a product that gives site owners full control to create and build their site.

The product consists on the themes and templates that we will create, and a “Prebuilt Demo Website” configured in the most common way.

In summary this is the process: we install the demo web site for site owners to use as a starting point; the site owners have full control to manage the content of their sites; we provide full support of our product to the site owners.

The main advantage of this solution is scalability. We can delegate the responsibility of creating and maintaining the websites to the site owners. For example, instead of us making 100 websites, we can have 100 site owners creating their own website. However, in order for this to work, we need a good product (themes/templates), and provide good guidance and support.

This is the implementation process:

Implementation Process Stage Parties involved Description

1. Build Product (Themes/Templates and Demo Website)

 Implementation Team

We build the themes and main templates.

We build a demo website that has the most used pages and elements. (e.g. home, about, services, resources, FAQs, and contact).

We only build this website once, and it will be used multiple times.

2. Export/Import Website

 Implementation Team

We export data from the demo site, and import it into the client (site owner) site.

3. Website Customization

 Site Owner

Site owner will personalize his web site. He won’t start the website from scratch since we installed the demo site. He will only need to edit/add/delete content.

These are the support resources we need to provide to the site owners:

Item Description

1. Text Documentation and Guidelines

Written documentation will show all the elements and functionality of our themes/templates, and explains how it implement it. For example:

  • What is a H1 and how to insert it.
  • What is a primary button or a secondary button and how to insert them.
  • What is a form and how to insert it.

Guidelines are best practices that the site owners should follow in certain scenarios. For example:

  • When it is appropriate to use a H1
  • When it is appropriate to use a primary button or a secondary button
  • When it is appropriate to use forms

2. Video Tutorials for common tasks

Viewing how to do something sometimes is the easiest way to understand it. We can make video demonstrations of common tasks such as:

  • How to create a new page
  • How to add a page title
  • How to insert a gallery
  • How to insert an accordion
  • How to delete a page

3. Demo Website

The Demo Website will show everything the site owner needs to see in order to replicate it:

  • Pages (home, about, resources, FAQs...)
  • Functionality (page layouts/templates, blog, responsiveness...)
  • Elements (navigation bar, secondary navigation, accordions, galleries, typography hierarchy...)

4. Forum
  • Site owners can post questions in the forum, and a theme developer (on our side) can respond these answers on daily basis.
  • The questions and answers will be archived and will be public. Eventually we will have the same questions asked several times, and we will be able to reuse previous answers.
  • We can mark FAQs. This way site owners can find those answers faster.

5. Stock photography and iconography.
  • This will allow the sites to look professional.
  • This will reduce time in creating custom graphics and photograpy (when applicable).

Pro Implementation (Website Created by Implementation Team)

This implementation process will be unique for websites that can’t be built with the previously described process, and that require custom design and development.

We will need implementation teams. The implementation teams should consist of: Project managers, designers, developers, and copywriters. One team member can have multiple roles. And a team member (e.g. copywriter) can be member of multiple teams.

The following is the implementation process:

Implementation Process Stage Parties involved Description
1. Planning Implementation Team & Site Owner
  • Most important step of the process
  • Define Requirements
  • Gather and Create Content
  • Build Wireframes or Prototype
  • Get Site Owner Approval
2. Design Implementation Team
  • Create high-fidelity mockups (only when needed).
  • Create visual design assets for the website.
  • Export visual assets.
3. Development Implementation Team
  • Develop special functionality for the website.
4. Feedback Implementation Team & Site Owner
  • Presentation of the site in a test environment.
  • Site Owner will provide one round of consolidated (everything in one single list) and documented (written in a document file) feedback. Limiting the rounds of feedback allow us to complete the project more efficiently and faster. This makes both parties happier. Also the product here is not a surprise for the site owner since in stage 1, the site owner approved either wireframes or a prototype and agreed on the requirements and functionality.
  • Will request approval from Site Owner to publish the site.
  • At this point the website will only work in one browser (Chrome most likely) since it has not been tested.
5. Testing Implementation Team
  • Test in supported browsers and devices (recommended support: IE7*, IE8*, IE9*, IE10*, IE11, Safari, Firefox, Chrome, iOS, and Android.
    *=Functional support only (it works properly but some visual elements are not supported - like animations, rounded corners, box-shadows...)
6. Site Launch Implementation Team
  • Publish the site in the production environment.
7. Go-Live QA Implementation Team
  • Extra round of testing to verify everything was published correctly
