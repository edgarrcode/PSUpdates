Crosswalk August 2013

Integration Testing To Start August 26

August 26 marks the beginning of the Integration Testing phase of the PeopleSoft project. As you may recall, Integration Testing is the phase of testing that follows System Testing. Unlike System Testing, where business processes were tested within each module, Integration Testing will test business process transactions as they occur across modules, between both HR and Finance. “PeopleSoft is different from DEFINE in that there’s a new level of interdependencies between the modules. Transactions that occur in one module might have downstream impact on a completely different module,” says Iris Niestas, PeopleSoft project manager at UTEP. “This is a new concept for many of us on campus, so we need to test the system to make sure that we understand how it works.”

Prepping For Integration

Prior to integration testing, the project team needs to make sure that entrance criteria are met so that testing can be successful. For example, team members are validating data, ensuring that system security is working properly, testing customizations to the system, and resolving open issues.

The SWAT Team Initiative

‘SWAT’ teams, comprised of select subject matter experts (SMEs) across all participating institutions, are currently working on testing security roles that will be used for integration testing. Security roles determine what level of access end-users will have to PeopleSoft. “The issues we’ve identified so far have to do with roles not having the right level of access in place,” says Lupe Gomez, Contracts and Grants subject matter expert (SME) and project team member assigned to the security SWAT team. “It may be that a role only has access to view information, when it should also have access to add/update information as well. We’ve been able to resolve all identified issues quickly with the help of security analysts assigned to assist during security testing.”

Payroll Parallel Testing

Payroll parallel testing is one vital component of integration testing that the project team is keeping a close eye on. During this time, three payroll runs will occur. The payroll runs will occur concurrently with Define payroll runs. Testing will verify that the payroll runs in both Define and PeopleSoft will yield the same payroll check amounts and deductions. “We really need to make sure that we’re testing the system vigorously,” said Iris. “This phase of testing will help us to ensure that all faculty, staff, and student payroll checks will be processed accurately at go-live.”
